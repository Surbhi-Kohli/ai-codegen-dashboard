Key Design Decisions

  • Near real-time ingestion -- webhooks for GitHub, Webex, git hook; polling for Jira only
  • Enrichment as a separate step -- raw events stored first, then joined and correlated
  • Single database -- all sources unified; no need for a data warehouse at this scale
  • API-first -- dashboard consumes the same API that could serve other tools later
